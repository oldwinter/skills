# Checklists

Use these to validate the Media Relations Pack before finalizing.

## A) Truth + safety checklist (required)
- [ ] No invented facts, metrics, customers, partnerships, or “exclusive interest”
- [ ] Confidential information is excluded or explicitly marked “not for sharing”
- [ ] High-risk/regulated claims are avoided or flagged for expert review
- [ ] If using embargo/exclusive, terms are clear and internally approved

## B) Newsworthiness checklist
- [ ] Clear “what’s new” (not just “we’re excited”)
- [ ] Clear “why now” tied to timing/context
- [ ] Reporter relevance: the angle fits a real beat
- [ ] Proof exists for key claims (or is labeled “to validate”)

## C) Targeting + list quality checklist
- [ ] Targets are tiered (1/2/3) with rationale
- [ ] Each target has a personalized hook
- [ ] List is sized to capacity (quality > volume)
- [ ] Relationship status and next actions are tracked

## D) Exclusive/embargo checklist (if used)
- [ ] Exclusive offer is explicit (what access/assets they get)
- [ ] Exclusive is timeboxed with a clear decision deadline
- [ ] Staggered outreach plan exists (wave 1, wave 2)
- [ ] Fallback plan exists if the exclusive declines or doesn’t respond

## E) Pitch quality checklist
- [ ] Pitch is short and specific (≤ ~150–200 words before signature)
- [ ] First sentence contains the news
- [ ] Includes 1–3 proof bullets (or “available on request”)
- [ ] Clear CTA and easy next step (call/demo/interview)
- [ ] No hype adjectives doing the work

## F) Materials readiness checklist
- [ ] Press materials checklist is complete (or gaps are explicit)
- [ ] Media FAQ exists with on-the-record answers
- [ ] Spokesperson bio and assets are ready (or placeholders exist)

## G) Execution + follow-through checklist
- [ ] Outreach tracker is populated and current
- [ ] Follow-up cadence is respectful and scheduled
- [ ] Interview prep includes 3 key messages + sensitive topics
- [ ] Post-coverage plan includes thank-yous, amplification, and relationship notes

## H) Finalization checklist
- [ ] Includes **Risks**, **Open questions**, **Next steps**
- [ ] Assumptions/TBDs are labeled
- [ ] Output is usable as-is with minimal editing
