# Skill Pack Quality Rubric (self-check)

Score each item 0/1/2:
- 0 = missing / unusable
- 1 = present but incomplete
- 2 = clear, executable, reusable

Suggested passing bar: >= 16/20. Below 14 = rework.

## 1) Discovery & triggering (0–2)
- Frontmatter description contains realistic trigger phrases (synonyms, artifacts)
- Clearly states when to use the skill

## 2) Boundaries (0–2)
- “When NOT to use” is explicit
- Includes escalation points for human decisions

## 3) Input contract (0–2)
- Minimal required inputs listed
- Missing-info strategy: ask max 3–5 questions, otherwise make assumptions

## 4) Output contract (0–2)
- Deliverables are explicit and checkable
- Output order is defined (if multiple artifacts)

## 5) Workflow clarity (0–2)
- 5–9 steps max
- Each step: inputs → actions → outputs → checks

## 6) Templates & artifacts (0–2)
- Templates exist for core deliverables
- Templates are copy/paste-ready (not abstract)

## 7) Quality gate (0–2)
- Checklist/rubric exists
- Final output always includes risks, open questions, and next steps

## 8) Examples (0–2)
- 2 positive examples + 1 boundary/negative example

## 9) Token efficiency (0–2)
- SKILL.md is lean; long content moved to references/
- No redundant prose; high signal density

## 10) Safety & robustness (0–2)
- Least privilege tool usage
- Clear data handling rules (no secrets, no exfiltration)
