## 2) Preboarding Checklist

**Logistics (complete before day 1)**
- [ ] Offer/HR paperwork done (owner: HR)
- [ ] Equipment ordered + delivered (owner: IT)
- [ ] Accounts created (email, SSO, VPN, repo, docs, analytics) (owner: IT)
- [ ] Permissions granted for essential systems (owner: manager)
- [ ] Calendar invites sent (team rituals, 1:1s, onboarding sessions) (owner: manager)
- [ ] Buddy assigned and briefed (owner: manager)
- [ ] “First pair” assigned and briefed (owner: manager)

**Context pack (sent before day 1 if possible)**
- [ ] Team operating manual + “how we work”
- [ ] Glossary (acronyms, tools, domain terms)
- [ ] Strategy/priorities doc
- [ ] Key metrics dashboard links
- [ ] Decision history (top 5 decisions + why)
- [ ] Reading list (optional)

## 3) First-Week Plan (day 1 + week 1)

**Day 1 (belonging + orientation)**
- Welcome + setup checklist (accounts, tools)
- Buddy time (tour + norms)
- First pair session (1–2 hours) on a real task
- Team introductions (live or async)
- “How we work” walkthrough (10–20 minutes)

**Week 1 (structured collisions)**
- 1:1 with manager (expectations + success definition)
- 1:1 with buddy (daily short check-in)
- Pairing blocks (2–4 sessions)
- Intro meetings with top stakeholders (3–6 meetings)
- First contribution: small, real deliverable (doc update, metric definition, bug fix, customer call summary)

---

## 4) Context Pack / Team Operating Manual Outline

1) Team mission + what we’re optimizing for
2) How we make decisions (inputs, owners, escalation, decision logs)
3) Product/engineering philosophy + mental models (manager + team)
4) What “good” looks like here (examples of great work)
5) Metrics: north star + leading indicators + dashboards
6) Cadence: rituals, ceremonies, meeting norms
7) Stakeholders: who we serve + key partners
8) Glossary: acronyms + tools + where truth lives

1) What are we optimizing for right now (and why)?
2) What’s working well today? What should we protect?
