# Checklists

Use these before finalizing the Launch Marketing Pack and before launch day.

## A) Launch brief checklist
- [ ] Clear audience/ICP and use case
- [ ] One-sentence message a teammate can repeat
- [ ] Concrete hook/sizzle (showable/measurable), not vague hype
- [ ] Proof points are real, substantiated, and safe to share
- [ ] Single CTA (what the audience should do next)
- [ ] Constraints and approvals are documented (legal/compliance/brand)

## B) PR outreach checklist (if using PR)
- [ ] Exclusive vs broad decision is explicit
- [ ] Target outlet(s)/reporters are a clear fit for the story
- [ ] Pitch email is short, specific, and includes 1–3 proof points
- [ ] Follow-up cadence is defined
- [ ] Website readiness confirmed (speed, signup flow, analytics)
- [ ] No confidential information included

## C) Channel + asset plan checklist
- [ ] Channel plan table filled (channel → asset → date → metric)
- [ ] Every channel row maps to the core message + hook
- [ ] Assets have owners and due dates
- [ ] Landing page and signup flow are tested

## D) Internal readiness checklist
- [ ] 30-second talk track exists
- [ ] FAQ covers top 10 questions
- [ ] Top objections and responses prepared
- [ ] Known limitations and guardrails stated
- [ ] Escalation path for edge cases is clear

## E) Measurement + experiments checklist
- [ ] Primary/secondary/guardrail metrics defined
- [ ] Instrumentation assumptions listed (and gaps called out)
- [ ] Experiment backlog includes thresholds and owners
- [ ] “Double down” rule is explicit

## F) Day-of checklist
- [ ] Final QA: site, signup, links, analytics, email deliverability
- [ ] Monitoring: traffic, conversion, error rates, support inbox
- [ ] Comms plan: internal updates cadence, who decides changes
- [ ] Backup plan: what to do if site breaks or feedback is negative

## Required final section
- [ ] Risks
- [ ] Open questions
- [ ] Next steps
