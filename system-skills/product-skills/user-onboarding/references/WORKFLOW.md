# WORKFLOW (User Onboarding → Activation)

Use this workflow to produce an **Onboarding & Activation Pack**. Copy/paste templates from `references/TEMPLATES.md`.

## Step 1 — Intake + goal framing
- Fill: **Onboarding Brief** template.
- Output: a measurable goal + guardrails + constraints.
- Gate: goal includes baseline → target + date.

## Step 2 — Map the current FTUE (first-time user experience)
- Fill: **FTUE Journey Map** + **Friction Log** templates.
- Output: the biggest leak + the top 5 frictions ranked by impact.
- Gate: each step has an event/proxy metric.

## Step 3 — Define activation (“aha moment”) and validate
- Fill: **Activation / Aha Moment Spec** template.
- Output: a single activation definition + threshold + validation plan.
- Gate: activation is behavioral + measurable.

## Step 4 — Design the first 30 seconds (fast, interactive win)
- Fill: **First 30 Seconds Spec** template.
- Output: the exact first-run script (what the user sees/does) + success criteria.
- Gate: “meaningful win” in ~30 seconds (or best proxy clearly stated).

## Step 5 — Build the first mile plan (milestones → activation)
- Fill: **First Mile Plan** template.
- Output: 3–6 milestones, each with mechanics + leading-indicator metrics.
- Gate: milestones reduce time-to-value; no “tour-only” steps.

## Step 6 — Instrumentation + measurement
- Fill: **Measurement Plan** template.
- Output: event schema + funnel + dashboard outline + guardrails.
- Gate: every milestone and experiment has primary + guardrail metrics.

## Step 7 — Experiment backlog + rollout plan + quality gate
- Fill: **Experiment Backlog** + **Experiment Card** + **Rollout/Rollback Plan** templates.
- Output: prioritized experiments and a release plan.
- Gate: run `references/CHECKLISTS.md`, score `references/RUBRIC.md`, and include **Risks / Open questions / Next steps**.
