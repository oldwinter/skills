# RUBRIC (User Onboarding Skill Output)

Score each item 0/1/2:
- 0 = missing / unusable
- 1 = present but incomplete
- 2 = clear, executable, reusable

Suggested passing bar: **>= 16/20**.

## 1) Goal clarity (0–2)
- The goal includes baseline → target and a date.
- Guardrails are explicit.

## 2) Segmentation (0–2)
- 1–2 segments are defined and the plan is tailored to them.

## 3) Journey map + friction log (0–2)
- FTUE steps are mapped end-to-end with measurable checkpoints.
- Top frictions are ranked and tied to fixes.

## 4) Activation definition (0–2)
- Activation/aha moment is behavioral, measurable, and tied to user value.

## 5) First 30 seconds quality (0–2)
- The “first win” is concrete and achievable quickly (or proxy is justified).
- Design avoids passive tours and focuses on interactive value.

## 6) First mile plan quality (0–2)
- Milestones are specific, realistic, and reduce time-to-value.
- Each milestone includes mechanics + metrics.

## 7) Measurement plan (0–2)
- Minimum viable event schema is defined.
- Dashboards and owners (if known) are specified.

## 8) Experiments (0–2)
- Backlog is prioritized with a clear scoring model.
- Top experiments have “win/lose/learn” criteria and rollout/rollback plans.

## 9) Safety + ethics (0–2)
- Avoids dark patterns; respects privacy; no secrets/PII requested.

## 10) Finalization (0–2)
- Includes **Risks**, **Open questions**, and **Next steps**.
- Output is structured, copy/paste-ready, and high signal.
