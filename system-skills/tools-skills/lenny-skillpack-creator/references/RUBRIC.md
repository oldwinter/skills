# RUBRIC (definition of done)

Score each category 0–2. Total /12.

## 1) Boundary clarity
- **0**: Vague; overlaps everything.
- **1**: Basic when-to-use, but weak “when NOT to use”.
- **2**: Clear triggers + clear exclusions + points to adjacent skills.

## 2) Input contract
- **0**: Missing or assumes perfect information.
- **1**: Lists inputs but unclear missing-info strategy.
- **2**: Minimum inputs + 3–5 question batching + explicit assumptions when missing.

## 3) Output contract
- **0**: No explicit deliverables.
- **1**: Deliverables exist but not ordered or not concrete.
- **2**: Ordered, explicit artifacts (files/sections) + always includes risks/open questions/next steps.

## 4) Workflow executability
- **0**: “Think/consider” steps; no operational flow.
- **1**: Steps exist but checks are vague or missing.
- **2**: 5–9 steps; each has Inputs/Actions/Outputs/Checks.

## 5) Quality gate
- **0**: No checklist/rubric.
- **1**: Checklist exists but not tied to outputs.
- **2**: Checklist + rubric used as a final gate; clearly callable.

## 6) Examples + testability
- **0**: No examples.
- **1**: Examples exist but don’t specify outputs or boundaries.
- **2**: 2 good prompts + 1 boundary prompt; outputs are explicit and verifiable.

## Pass threshold
- **Pass**: ≥10/12 and no category scored 0.
- **Ship/no-ship**: If any category is 0 → fix before merging.
