# Checklists

## A) Preboarding readiness
- [ ] Access + tools are ready (SSO/VPN/repos/docs/dashboards)
- [ ] Buddy and first pair assigned (and understand responsibilities)
- [ ] Manager has a 30/60/90 draft ready (even if rough)
- [ ] Calendar is populated (rituals + 1:1s + intro meetings)
- [ ] Context pack exists (or a plan to create it in week 1)

## B) Day 1 / week 1 belonging
- [ ] The hire is not left alone on day 1 (buddy + first pair time scheduled)
- [ ] Introductions cover immediate collaborators and key interfaces
- [ ] A small real contribution is identified for week 1
- [ ] The hire knows where to ask questions and how decisions get made

## C) Listening tour quality (0–30)
- [ ] Stakeholder map includes all critical interfaces (cross-functional + exec as needed)
- [ ] Meetings are timeboxed and scheduled (not “sometime soon”)
- [ ] Questions force specifics (examples, metrics, trade-offs)
- [ ] Notes are synthesized into themes + evidence (not a transcript)
- [ ] A 30-day state-of-the-union memo is produced and shared appropriately

## D) Working agreement quality (relationship design)
- [ ] Expectations are explicit (what you need from them, what they need from you)
- [ ] Communication norms are agreed (channels, response times, escalation)
- [ ] Decision rights are clarified (who decides what)
- [ ] Feedback cadence is defined (how and when feedback happens)

## E) 30/60/90 plan quality
- [ ] Each phase has objectives, deliverables, and measurable checks
- [ ] The plan is realistic for seniority + context (not a “hero plan”)
- [ ] Guardrails exist (“what not to change yet”)
- [ ] Stakeholder checkpoints are scheduled (alignment moments)
- [ ] 1-year/2-year expectations are captured (even if rough)

## F) Final pack completeness
- [ ] Includes **Risks / Open questions / Next steps**
- [ ] Assumptions are labeled; missing info is listed
- [ ] Outputs are calendar/doc-ready (actionable, not generic advice)
